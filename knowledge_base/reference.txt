# EPM Error Notebook (sample — replace with your own)

## FDMEE: "No periods were identified for loading data"
**Seen in:** file-based data load, ACTUALS rule
**Root cause:** the source period(s) in the file have no row in the *Application*
period mapping, or the Global period mapping is missing the calendar entry. Often a
new month was opened but never added to Period Mapping.
**Fix:**
1. Data Management > Setup > Period Mapping > Global Mapping — confirm the period exists.
2. Application Mapping for the target app — map the source period to the target.
3. Re-run; if still failing, check the POV period matches the file.
**Confirmed by:** load shows rows imported > 0 in Workbench.

## FDMEE: period mapping null end date
**Root cause:** a period row exists but the Target Period Month / end date column is blank.
**Fix:** edit the period row, set the period start/end and target period month, save.

## EPBCS: "No mapping found for source member X in dimension Y"
**Root cause:** the data load mapping for that dimension has no rule matching the source value.
**Fix:** Data Load Mapping > select the dimension > add an Explicit or Like (wildcard `*`)
rule mapping the source to a valid target member. Verify the target member exists and is level-0.

## ODI session failed + ORA-12899 (value too large for column)
**Root cause:** a source value is wider than the target/staging column — usually an
amount or a text member name exceeding the column length.
**Fix:** trim/clean the offending source value, or check the import format amount column
mapping. Inspect the ODI Operator session log for the exact column.

## IDCS OAuth2 AUTH-1163
**Root cause:** OAuth2 client/app config mismatch or expired token for the EPM REST call.
**Fix:** re-validate the IDCS confidential app scopes and client secret; regenerate the
token; confirm the role/grant on the EPM Cloud instance.

Error: Validation error with missing mappings
Resolution:
Mappings were missing in one of the dimensions. Navigate to Workbench → Validation Errors and use the Fix Mappings option to create or update the required mappings.

Error: 401 Unauthorized while starting the Integration Agent
Resolution:
The encrypted password may be incorrect, or the user does not have the required access/roles. Verify the encrypted password configuration and confirm the user has appropriate access to the environment/application.

Error: Plan Type Null
Resolution:
In the Data Load Rule options, ensure a valid Plan Type/Cube is selected. If the Plan Type field is left blank, the load process will fail with this error.

Error: Non Numeric Data
Resolution:
Set the delimiter in the Import Format to All Data Type. Incorrect delimiter settings can cause numeric fields to be interpreted improperly during the load process.

Error: Invalid Period Mappings
Resolution:
The period format in the source file is incorrect or does not match the configured format. Verify the source period format and update the mappings in Source Period Mappings accordingly.

Error:
com.microsoft.sqlserver.jdbc.SQLServerException: "encrypt" property is set to "true" and "trustServerCertificate" property is set to "false" but the driver could not establish a secure connection to SQL Server by using Secure Sockets Layer (SSL) encryption: Error: PKIX path building failed.
Resolution:
This is an SSL certificate trust issue between the Microsoft SQL Server and the EPM Integration Agent Java process. Update the JDBC URL to include:
encrypt=true;trustServerCertificate=true;
Example:
jdbc:sqlserver://Server:Port;databaseName=DBName;encrypt=true;trustServerCertificate=true;

Error: 3303 Member Missing in EPM
Resolution:
The member does not exist in the target application. Create the missing member in the target application or update the mapping to ignore the member if it is not required.

Error: Unknown Member Name
Resolution:
Use the fully qualified member name in the filters. Incomplete or incorrect member references can cause this error during execution.

Error: The current 'Smart List/Pivot Dimension' does not contain smart list data type members. Change the 'Smart List/Pivot Dimension'.
Resolution:
When using Data Maps to map Smart List values to dimension members across cubes, verify the selected Pivot Dimension. Navigate to Data Map Actions → Options and select the correct Pivot Dimension containing Smart List data type members.

Error: Cannot load data when active users or AIF processes are running
Resolution:
This error occurs when another Data Management rule, refresh, or backend AIF process is already running. Wait for the active process to complete and then rerun the data load.
